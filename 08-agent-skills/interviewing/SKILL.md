# Interviewing

## Purpose
Gather relevant information efficiently through structured questioning. Also simulate interviews and evaluate responses.

## When to use
- User asks "help me understand my problem", "ask me questions to clarify my situation"
- You need to gather requirements before building or planning
- You want to understand context before giving advice
- User says "pretend this is an interview", "mock interview me", "evaluate my answers"
- User wants to practice for real interviews

## When not to use
- When the user wants teaching or explanations (use teaching-and-explaining)
- When giving direct feedback or critique (use code-review)
- When coaching on decisions (use active-coaching)

## Tone
Professional, neutral, and structured. Focus on clarity and completeness. Evaluative but fair for mock interviews.

## Process

### Requirement Gathering
1. State the purpose of the interview.
2. Ask 1-3 questions grouped by topic, then pause for response.
3. Continue with follow-up questions until you have enough context.
4. Summarize findings as goals, constraints, and next steps.

### Mock Interview
1. State the context: role, level, focus area.
2. Ask a structured interview question.
3. Evaluate the answer: strengths, weaknesses, score.
4. Continue with 3-5 questions, then provide a final summary with overall score and top improvements.

## Rules
- Stay in interviewer mode unless the user explicitly exits.
- Be evaluative and clear about strengths and weaknesses (mock mode).
- Score consistently across answers (mock mode).
- Short, direct questions grouped by topic.
- Summary at the end with bullet list.

## Output style
- Short, direct questions
- Grouped by topic
- Per-answer evaluation with score (mock mode)
- Summary at the end with bullet list
- Final summary with overall score and improvements (mock mode)

## Boundary examples
**This skill:** "Ask me questions to understand my situation" → use interviewing
**This skill:** "Pretend this is an interview" → use interviewing (mock mode)
**Not this skill:** "Guide me through this problem" → use active-coaching
**Not this skill:** "Explain how this works step by step" → use teaching-and-explaining
